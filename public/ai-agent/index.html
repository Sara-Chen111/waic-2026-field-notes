第一部分 · 我使用的工具是什么

AI 正从"回答问题",
变成执行任务。

— 1. 从大模型到 Agent —

过去我们使用 AI，更多是在网页或聊天窗口里提问，让它给答案、文案、代码或思路——这个阶段的 AI 更像一个"高级问答工具"。

现在头部 AI 公司在推动的，是把大模型进一步做成 Agent 工具：不只回答，而是能根据目标拆解任务、调用工具、读写文件、执行命令，并在过程中不断调整方案。

换句话说，它的能力从"给建议"升级为"帮我把事情做出来"。这次分享的核心，不是某个 AI 模型有多聪明，而是如何把 AI 变成一个可以参与日常工作的执行型助手。

— 2. 头部 AI 公司都在做自己的 Agent 工具 —

全球最重要的几家 AI 大模型公司，除了持续升级基础模型，也在推出各自的 Agent 产品。要区分两个概念：大模型负责"思考和生成"，Agent 工具负责"组织任务和执行动作"。

Anthropic（An-thro-pic）
大模型：Claude 系列
→ Agent 工具：Claude Code
← 我用这个
OpenAI

→ Agent 工具：Gemini CLI
🦞
开源 / 社区工具 · OpenClaw（龙虾）
"个人 AI Agent 平台，可接入不同模型。"
OpenClaw 更像一个个人 AI Agent 平台，可以接入 Claude、GPT、Gemini 等模型，通过聊天工具、终端或网页执行任务。它的走红也说明：市场对"AI 真正帮我做事"的期待，已经非常强烈。
两个概念，一定要区分清楚
大模型（Claude Opus、GPT、Gemini、GLM）负责"思考和生成"；Agent 工具（Claude Code、Codex、Gemini CLI、OpenClaw）负责"组织任务和执行动作"。所以，Claude Code 本身不是模型，而是把模型能力连接到真实工作流的执行工具。

第三部分 · ARB Agent 的工作计划

从"AI 帮我做"
到一个能参与业务流程的工作系统。

1
PHASE 1 · 进行中
提出需求，让 AI 执行
人提需求，AI 跑执行。先找到——哪些工作能交给 AI 做 70%，由人做最后判断。
  • 数据整理 / 文件批量处理
  • 页面生成 / 文案初稿
  • 表格优化 / 简单规则判断
2
PHASE 2
高频工作沉淀成脚本
反复验证过的任务，不必每次让 AI 从头思考。稳定、重复、规则清晰的工作沉淀成脚本。

① 降 token 消耗；
② 提高稳定性。

分工：规则交给脚本，判断交给 AI。
3
PHASE 3
加入主观判断类任务
逐步尝试图片打标、素材筛选、质量判断。主观判断、品牌风格容易偏差，靠训练和校准持续靠近业务标准：
  • 建立样例 → AI 按规则打标
  • 人工复核 → 找出错误类型
  • 反向优化提示词与规则
— 4. 最终希望形成的工作模式：三层能力 —

未来 ARB Agent 的目标不是做一个"聊天机器人"，而是形成一个能参与实际业务流程的工作系统。它应该具备三层能力：

第一层
脚本自动化
稳定 · 低成本 · 可重复
批量处理、格式转换、字段清洗、基础检查
第二层
AI Agent 执行
能理解需求并调用工具
页面生成、数据整理、复杂文件处理、跨步骤任务
第三层
AI 辅助判断
能参与主观判断 · 需要训练
图片打标、素材筛选、质量判断、风格判断
核心思路
我们不是把所有事情都交给 AI，而是把工作拆成不同层级：能脚本化的脚本化，需要判断的让 AI 辅助，人保留最终判断权。
— FINAL REEL · 这次实践给我的启发 —

人把目标讲清楚，
AI 把过程跑起来。

AI Agent 的价值，不只是"生成内容更快"，
而是改变工作分工——让人更专注于判断什么值得做、什么是好结果。

I
过去工作卡在哪里

想法有了，但没有足够时间把它做出来。

过去很多工作卡在两个地方——想法有了，但没时间把它做出来；需求知道，但实现过程太零散、太耗人力。

现在 Agent 工具让这件事发生变化：人把目标讲清楚，AI 把过程跑起来。我们可以用更低成本验证更多想法，用更短周期完成更多基础工作，把人的精力更多放在判断、标准和方向上。

II
分工的转变

我负责判断什么值得做，AI 负责把它先做出来。

在我的实践中，分工已经发生了清晰的变化——

✦ 人负责
  • 判断什么值得做
  • 把业务目标讲清楚
  • 对结果做最终判断和方向校准
✦ AI 负责
  • 拆解任务、调用工具、读写文件
  • 执行命令、生成内容、修正错误
  • 在过程中不断调整方案
III
ARB Agent 的定位

不是替代人，而是把人从重复执行中释放出来。

所以，我对 ARB Agent 的定位是——它不是替代人，而是把人从重复执行中释放出来，让人更专注于判断什么值得做、什么是好结果。

FIN.
人把目标讲清楚，
AI 把过程跑起来。
